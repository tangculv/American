# 美股智能选股研究系统 — 开发执行计划

## 一、总体策略

### 核心原则
- **PRD 唯一基线**: `PRD-开发基线版.md` v3.0-baseline.1 是唯一开发依据
- **AI 开发 + AI 验证 + AI 测试**: 全程由 AI 执行，人只审批关键节点
- **分 Phase 交付**: 4 个 Phase，每个 Phase 内含开发→自验→测试→交付
- **SQLite 唯一事实源**: 所有业务数据先写 DB，JSON/MD 只做导出

### 角色定义

| 角色 | 职责 | 执行者 |
|------|------|--------|
| AI 开发 | 按 PRD 写代码、建表、写配置 | Claude/GPT |
| AI 验证 | 代码写完后自查: 字段是否与 PRD 一致、接口是否完整、审计是否覆盖 | Claude/GPT |
| AI 测试 | 运行单元测试 + 集成测试 + 链路回放测试 | Claude/GPT |
| 人工审批 | Phase 交付物审批、买卖决策审批、复盘变更审批 | chengxiaoming |

### 每个 Phase 的标准工作流

```
1. AI 开发 → 输出代码 + 配置
2. AI 验证 → 对照 PRD 逐字段自查 → 输出验证报告
3. AI 测试 → 运行测试套件 → 输出测试报告
4. 人工审批 → 确认交付物 → 进入下一 Phase
```

## 九、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| FMP API 限流 | 高 | 筛选延迟 | 缓存+降频+夜间执行 |
| Perplexity 返回格式不稳定 | 中 | 研究解析失败 | JSON 校验+fallback+手动标记 |
| 飞书 Webhook 不稳定 | 低 | 通知延迟 | 重试+日志+每日摘要兜底 |
| SQLite 并发写入冲突 | 中 | 数据丢失 | WAL 模式+写入锁+重试 |
| 评分公式调整频繁 | 低 | 历史不可比 | formula_version + 不回溯重算 |
