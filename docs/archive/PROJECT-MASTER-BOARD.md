# 项目总盘

> 目标：解决“做了很多，但用户感知不到整体进展”的问题。  
> 基线：`PRD-开发基线版.md`

本文件不是新的 PRD，而是把当前项目按 **整体设计 + 分环节进度** 重新编排成一个用户可直接阅读、可随时核验的总盘。

## 1. 你现在最需要知道的结论

当前项目不是停留在“代码开发中”的状态，而是已经形成了一个 **可直接运行的基线闭环产品**：

- 可做真实筛选
- 可做研究与评分
- 可推进生命周期
- 可执行买卖记录与持仓跟踪
- 可发送通知
- 可做复盘审批
- 可通过 UI / CLI 直接使用

但它**还没有完全达到“所有 PRD 细则 100% 封口”**。当前更准确的状态是：

> **主干闭环已可用，关键模块大多可直接使用，正在继续补齐总盘可视化、治理细则、自动化收口。**

## 2. 总盘设计：整体信息架构

为了让进展可感知，项目总盘按 12 个一级模块管理：

1. 数据源与配置层
2. 策略筛选层
3. 生命周期状态机
4. 深度研究（Perplexity）
5. 综合评分
6. 技术面与交易闸门
7. 排序与研究队列
8. 持仓跟踪与交易动作
9. 通知系统
10. 复盘与审批
11. 调度与自检
12. UI / CLI 产品形态
13. 测试、回归与交付文档

这些模块共同对应基线 PRD 的主骨架：
- §1 系统目标与边界
- §2 端到端主流程

## 7. 当前主要风险

1. 外部 API 可用性受第三方网络和凭证影响

3. 技术面和评分模型仍有继续收口空间
