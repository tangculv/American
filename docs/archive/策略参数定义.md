## 🎯 STRAT_001：低估值高质量科技股

### 筛选条件

| 参数组 | 参数名 | 参数类型 | 当前值 | 说明 |
|--------|---------|----------|---------|----------|
| 市值 | market_cap_min | Number | 500000000 | 最小市值 >5亿 |
| 市值 | market_cap_max | Number | 100000000000 | 最大市值 <1000亿 |
| 成交量 | volume_min | Number | 1000000 | 最小成交量 >100万 |
| 行业 | sector | String | Technology | 科技股 |
| 交易所 | exchange | String | NASDAQ | 纳斯达克 |
| 排除ETF | is_etf | Boolean | false | 排除ETF |
| 排除基金 | is_fund | Boolean | false | 排除基金 |

### 评分权重

| 评分维度 | 权重 | 说明 |
|---------|------|------|----------|
| 估值（PE、PB） | 0.40 | 40% |
| 盈利能力（ROE、净利率） | 0.30 | 30% |
| 财务健康（负债率、流动比率） | 0.20 | 20% |
| 市值 | 0.10 | 10% |

**总权重**: 1.00

### 评分公式

```
总分 = 估值分数 + 盈利能力分数 + 财务健康分数 + 市值分数

估值分数 = 0
if PE > 0 and PE < 15:
    估值分数 += 40 * (1 - PE / 15)
elif PE > 0 and PE < 25:
    估值分数 += 20 * (1 - PE / 25)
elif PE > 0 and PE < 30:
    估值分数 = 20 * (1 - PE / 25)
else:
    估值分数 = 0

if PB > 0 and PB < 2:
    估值分数 += 10  # PB<2额外加分
elif PB > 0 and PB < 5:
    估值分数 = 0
else:
    估值分数 = 0

# ROE直接从API获取（优先使用roeRatioTTM字段）
if roe > 0.20:
    盈利能力分数 += 30  # ROE>20%加分30分
elif roe > 0.10:
    盈利能力分数 += 20  # ROE>10%加分20分
elif roe > 0:
    盈利能力分数 = 0

# 净利率补充（如果ROE不可用）
if 净利率 > 0.20:
    盈利能力分数 += 10  # 净利率>20%额外加分10分
elif 净利率 > 0.15:
    盈利能力分数 += 5  # 净利率>15%加分5分
elif 净利率 > 0:
    盈利能力分数 = 0

# 财务健康
if 负债率 > 0 and 负债率 < 1.0:
    财务健康分数 += 20  # 负债率<1优秀
elif 负债率 > 0 and 负债率 < 1.5:
    财务健康分数 += 15  # 负债率<1.5良好
elif 负债率 > 0 and 负债率 < 2.0:
    财务健康分数 += 5  # 负债率<2一般
else:
    财务健康分数 = 0

if 流动比率 > 1.5:
    财务健康分数 += 10  # 流动比率>1.5优秀
elif 流动比率 > 1.2:
    财务健康分数 += 5  # 流动比率>1.2良好
else:
    财务健康分数 = 0

# 市值
if 市值 > 10000000000:  # >1000亿
    市值分数 += 10
elif 市值 > 5000000000:  # >500亿
    市值分数 += 5
else:
    市值分数 = 0
```

## 🎯 STRAT_002：Fallen Angel（完整版）

### 策略说明

**目标**：寻找从高位跌落的优质公司

**核心指标**：
- 近1年跌幅：-60% 到 -10%
- 低估值：PE < 25、PB < 5
- 高质量：ROE > 15%、财务稳健

### 筛选条件

| 参数名 | 类型 | 当前值 | 说明 |
|--------|---------|----------|----------|----------|
| 市值 | market_cap_min | Number | 500000000 | 最小市值 >5亿 |
| 市值 | market_cap_max | Number | 100000000000 | 最大市值 <1000亿 |
| 成交量 | volume_min | Number | 1000000 | 最小成交量 >100万 |
| 行业 | sector | String | Technology | 科技股 |
| 交易所 | exchange | String | NASDAQ | 纳斯达克 |
| 排除ETF | is_etf | Boolean | false | 排除ETF |
| 排除基金 | is_fund | Boolean | false | 排除基金 |

### 评分权重

| 评分维度 | 权重 | 说明 |
|---------|------|------|----------|
| 跌幅 | 0.35 | 近1年跌幅是核心 |
| 估值 | 0.25 | PE、PB权重 |
| 盈利能力 | 0.25 | ROE、净利率 |
| 财务健康 | 0.15 | 负债率、流动比率 |

**总权重**: 1.00

### 评分公式

```
总分 = 跌幅分数 + 估值分数 + 盈利能力分数 + 财务健康分数

跌幅分数计算（需要历史价格）
涨幅 = (当前价格 - 52周前价格) / 52周前价格
if 跌幅 < -0.10:  # 跌幅>-10%
    跌幅分数 = 35  # 满分（35%权重）
elif 跌幅 < -0.20:  # 跌幅>-20%
    跌幅分数 = 30  # 良好（30%权重）
elif 跌幅 < -0.30:  # 跌幅>-30%
    跌幅分数 = 25  # 较好（25%权重）
elif 跌幅 < -0.40:  # 跌幅>-40%
    跌幅分数 = 20  # 一般（20%权重）
elif 跌幅 < -0.60:  # 跌幅>-60%
    跌幅分数 = 10  # 偏浅（10%权重）
else:
    跌幅分数 = 0
```

### API依赖

| 功能 | 状态 | 说明 |
|--------|------|------|----------|
| 基础筛选 | ⚠️ 可用 | FMP历史价格API不支持直接筛选跌幅 |
| 历史价格获取 | ❌ 不可用 | 需要使用其他数据源 |
| 财务比率 | ✅ 可用 | /stable/ratios-ttm |

### 实施流程

```
# 步骤1：基础筛选
candidates = fmp.screener(
    marketCapMoreThan=500000000,
    sector="Technology",
    exchange="NASDAQ",
    isEtf=False,
    isFund=False,
    limit=50
)

# 步骤2：获取财务比率
for stock in candidates:
    ratios = fmp.ratios_ttm(stock.symbol)
    pe = ratios.get('priceToEarningsRatioTTM')
    pb = ratios.get('priceToBookRatioTTM')
    roe = ratios.get('roeRatioTTM') or netProfitMargin / (1 - netProfitMargin)

# 步骤3：获取历史价格（需其他API）
for stock in candidates:
    prices = alpha_vantage.get_daily_prices(stock.symbol)
    跌幅 = calculate_1y_change(prices)

# 步骤4：综合评分
for stock in candidates:
    分数 = 跌幅分数 + 估值分数 + 盈利能力分数 + 财务健康分数
    排序选出Top 3
```
