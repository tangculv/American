# 美股选股体系 PRD（业务版）

## 1. 产品定义

一套本地运行的美股研究辅助系统。

系统负责：每天自动从全市场筛选股票、自动做深度研究、把结果推到飞书。
用户负责：看研究报告、自己决定买不买、录入买入信息。
买入后系统负责：帮用户盯着持仓，做风险控制和卖出机会提醒。

> 一句话：**系统帮你发现机会和盯盘，你来做决策。**

---

## 2. 核心使用场景

### 场景一：每日开盘自动推送

美股开盘后 5 分钟，系统自动完成以下动作：
1. 按策略从全市场筛选符合条件的股票
2. 对每只股票生成初步判断（关键指标 + 评估结论）
3. 检查 10 天内是否已做过深度研究
   - 已做过 → 跳过，在通知中标注"X月X日已研究，报告链接：xxx"
   - 未做过 → 自动发送 Perplexity 深度研究
4. 深度研究完成后，为每只股票生成飞书文档
5. 通过飞书通知用户："今日 N 只股票进入目标范围"

**用户体验**：每天早上打开飞书，就知道今天有哪些股票值得关注，点开链接就能看详细报告。

### 场景二：阅读报告做决策

用户在飞书中逐个打开研究报告。每份报告帮用户回答 6 个关键问题：
1. 这家公司到底有没有真家底？（有形净资产）
2. 报表利润有多少水分？（归一化盈利）
3. 当前价格买入，真实收益率大概是多少？
4. 安全边际来自哪里？（资产、现金流、还是两者兼有）
5. 现在值不值得买？
6. 如果买，怎么买、买多少、错了怎么退？

用户读完报告后的决策路径：
- **值得投** → 在网页看板录入买入信息，系统进入持仓跟踪
- **暂时不买但值得关注** → 保持"关注中"状态，后续命中时继续推送
- **不感兴趣** → 标记"已忽略"，不再通知和研究

**系统不替用户做买入决策。**

### 场景三：在网页看板管理

用户打开后台网页，可以看到：
- **候选池**：所有被筛选命中过的股票及其状态
- **持仓**：所有已买入股票的实时跟踪
- **策略配置**：配置和调整筛选策略

### 场景四：买入后的风险控制（核心场景）

用户标记买入后，系统自动进入持仓跟踪模式：
- 每日刷新价格、收益率、技术面状态
- 出现风险信号时飞书通知（如价格暴跌、技术面转弱、基本面恶化）
- 出现卖出机会时飞书通知（如达到目标价、技术面出现顶部信号）

用户的核心诉求是：**打开飞书或网页，5 分钟内知道持仓是否安全、是否需要操作。**

不需要操作时，系统不打扰。需要关注时，信号清晰、建议明确、一眼能看懂。

## 3. 产品功能定义

---

### 3.1 每日自动发现与研究

#### 功能目标
每个交易日自动完成"筛选 → 研究 → 推送"全链路，用户无需手动触发。

#### 触发方式

系统支持三种触发方式，去重规则各不相同：

| 触发方式 | 说明 | 是否走 10 天去重 |
|---------|------|-----------------|
| 定时自动运行 | 美股开盘后 5 分钟自动执行 | 是 |
| 手动运行全流程 | 用户手动触发一次完整筛选+研究批次 | 是 |
| 手动指定单只股票 | 用户指定某只股票立即研究 | 否（强制执行） |

> 开盘后 5 分钟运行，目标是尽早发现早盘进入目标范围的股票。后续可根据实际效果优化时点。

#### 全链路流程

**Step 1：全市场策略筛选**
- 根据用户配置的策略筛选全市场股票
- 对筛选结果按策略评分排序
- 输出候选股名单

**Step 2：初步判断**
- 对每只候选股生成初步评估：
  - 关键估值指标（PE、PB、PS、PEG）
  - 关键质量指标（ROE、净利率、负债率）
  - 关键市场数据（市值、成交量、涨跌幅、距52周高点跌幅）
  - 策略命中原因
  - 初步结论（一句话）

> **初步判断仅用于排序与摘要展示，不参与"是否进入深度研究"的决策。** 具体用途：候选池排序、飞书通知中每只股票的一句话摘要、当日研究名额分配的排序依据。后续版本可扩展为研究优先级排序。

**Step 3：去重判断**
- 查询该股票最近一次深度研究的完成日期
- 10 天内已做过 → 标记为"研究复用"，附带研究日期和飞书文档链接
- 超过 10 天或从未研究过 → 进入深度研究队列

10 天去重口径定义：
- **10 天 = 10 个自然日**（非交易日）
- 以**美东时区日历日**为准
- 判定方式：`今日美东日期 - 上次研究完成的美东日期 ≤ 10` 则复用
- 例：3月1日研究完成，3月11日仍可复用，3月12日需重新研究

去重例外：
- **持仓股**触发显著变化（单日价格变动 ≥ 5%、技术面质变）时，**可打破 10 天复用**，强制重研究
- **非持仓股**严格执行 10 天去重，无例外
- 用户手动指定单只股票研究时不受去重约束

研究复用的全渠道展示：
- **飞书通知**："AAPL: X月X日已研究（今日再次命中）→ [查看报告]"
- **网页候选池**：研究状态显示"复用(X月X日)"，同时显示"今日命中"标记

**Step 4：自动深度研究**
- 对需要研究的股票，按初步判断得分从高到低排序
- **当日最多研究 15 只**（含手动指定的单只股票）
- 超出 15 只的部分标记为"待研究"，保留在候选池中，次日批次优先处理
- Perplexity 研究请求需注意并发控制，避免触发限流
- 研究规格详见附录 A

**Step 5：生成飞书文档**
- 每只完成深度研究的股票，生成一份飞书文档
- 文档包含完整研究报告

**Step 6：飞书通知**
- 发送汇总通知到飞书：今日命中股票数、每只股票代码 + 一句话结论 + 报告链接
- 超出研究名额的股票在通知中标注"待研究，已入候选池"
- 失败/降级情况的展示规则见 3.7 节

#### "已忽略"股票的处理

已被用户标记为"已忽略"的股票，再次命中策略时：
- 候选池**仍记录命中**（更新最近命中日期、增加命中次数）
- **不发飞书通知**
- **不触发深度研究**
- 候选池中可看到"今日命中"标记
- 用户可随时手动将状态恢复为"关注中"

#### 产品规则
- 筛选结果必须持久化，每次运行有批次标识
- 筛选/研究失败必须有明确错误记录
- Perplexity 不可用时有降级行为（见附录 A）
- 去重周期默认 10 天，后续可配置
- 每日研究上限默认 15 只，后续可配置

### 3.2 飞书交付

#### 功能目标
把系统产出以飞书文档和飞书消息两种形式交付给用户。

#### 飞书文档（研究报告）

每份研究报告生成为一篇飞书文档，分为两个交付阶段：

**第一阶段（当前版本交付门槛）**

飞书文档至少包含以下内容，缺一不可：
1. 极简综合要点表（关键数据一张表看完）
2. 3 句话结论（真实资产底线、真实盈利能力、买入逻辑与最大风险）
3. 看多逻辑与看空逻辑
4. 关键风险与催化剂
5. 估值观点与目标价区间
6. 投资结论与操作建议（建仓区间、止损条件）

**第二阶段（后续增强，质量目标）**

以下内容属于完整投研模板的深度内容。Perplexity 能稳定输出则包含，不能稳定输出时**不阻塞第一阶段交付**，但需在文档中标注"部分内容缺失"：
- 归一化盈利桥接表
- 资产结构拆分表 + 有形净资产计算
- 负债与杠杆详细表
- 行业竞争对手对比表
- 三情景估值完整表
- 完整交易计划表

> **交付标准澄清**：第一阶段 6 项内容 = 交付门槛（必须达到才能推送文档）。附录 A 中的质量校验 = 质量目标（未达标时文档仍可交付，但标记"部分内容缺失"）。两者不冲突：门槛保证可用，目标追求完整。

产品规则：
- 文档标题格式：`[股票代码] 公司名 - 深度研究报告 (日期)`
- 降级研究文档标题加注 `[降级]`
- 文档链接必须可点击打开
- 文档创建失败必须有错误记录

#### 失败与降级时的用户展示

| 状态 | 飞书通知展示 | 候选池研究状态 |
|------|-------------|--------------|
| 研究成功 | 正常通知 + 文档链接 | 已研究(日期) |
| 降级研究 | 标注"[降级研究]" + 文档链接 | 降级研究(日期) |
| 研究失败 | 标注"研究失败，仅保留初步判断" | 研究失败 |
| 部分成功 | 汇总通知中逐只标注状态 | 各自对应状态 |
| 全批次失败 | "今日命中X只，全部研究失败" + 错误摘要 | 全部标记研究失败 |

#### 飞书消息通知

通知类型和治理规则见 3.7 节（通知体验设计）。

### 3.3 买入管理

#### 功能目标
用户决定买入后，手动录入买入信息，系统开始跟踪。

#### 用户操作

用户在网页看板上标记某只股票为"已买入"，录入以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 股票代码 | 是 | 大写，如 AAPL |
| 买入价格 | 是 | 美元，支持小数 |
| 买入日期 | 是 | 默认当天 |
| 买入数量 | 是 | 股数 |
| 买入原因 | 否 | 自由文本 |

#### 未在候选池中的股票

**允许用户直接录入未被策略命中过的股票。** 系统行为：
- 自动将该股票加入候选池（来源标记"手动录入"，命中次数为 0）
- **不自动触发深度研究**（用户可手动指定研究）
- 股票立即进入持仓跟踪

#### 系统行为
- 买入记录写入数据库
- 股票自动进入持仓跟踪池
- 飞书通知用户确认"已录入买入：AAPL @ $185.50 × 100股"

#### 成本与收益计算口径

| 项目 | 口径 |
|------|------|
| 持仓成本 | **加权平均成本法**：总投入金额 / 总持仓股数 |
| 部分卖出的已实现盈亏 | （卖出均价 - 加权平均成本）× 卖出股数 |
| 未实现盈亏 | （当前价格 - 加权平均成本）× 剩余持仓股数 |
| 持仓天数 | 从**首次建仓日期**算起（加仓不重置） |
| 加仓后的成本更新 | 重新计算加权平均成本 |

#### 产品规则
- 同一股票可以多次买入（加仓），每次独立记录
- 买入记录不可删除，只能标记为"已卖出"
- 买入信息录入后立即生效，不需要审批

### 3.4 买后管理（核心功能）

#### 设计理念

买后管理是本系统的**核心价值所在**。用户在发现和研究阶段投入的时间有限（每天看看飞书通知），但买入后的风险控制和卖出时机把握直接影响投资结果。

系统的目标是：**让用户不盯盘也能安心持有，该操作时不会错过。**

核心原则：
- **安静时真安静**：没有异常就不打扰，用户不需要每天主动检查
- **报警时够清晰**：信号含义明确、建议动作具体、一眼能判断是否需要操作
- **能追溯能复盘**：每个预警的触发、响应、结果都有记录

---

#### 3.4.1 日常持仓监控

每个交易日自动刷新所有持仓股票的最新状态。

每只持仓股票的监控快照至少包含：
- 当前价格
- 相对买入价的涨跌幅
- 未实现盈亏（金额和百分比）
- 持仓天数
- 当日涨跌幅
- 成交量（与近期均量对比）

日常监控是后台静默执行的，只在发现异常时才触发预警通知。

**异常隔离**：对单只持仓在监控流程中的异常做隔离并记录日志，不影响其他持仓处理。任何环节（价格刷新、信号检测、预警管理等）的异常都必须被隔离，确保一只股票的失败不阻断整体监控。

---

#### 3.4.2 信号体系

所有监控信号分为两个级别：

| 级别 | 名称 | 核心语义 | 用户理解 |
|------|------|---------|---------|
| **关注级** | 风险预警 | 风险上升了，需要关注 | "这只股票出现异常，我要盯紧它" |
| **动作级** | 卖出提醒 | 已达到动作条件，考虑操作 | "到了该考虑卖的时候了" |

归属原则：
- 趋势恶化但未触及动作阈值 → 风险预警（关注级）
- 已触及明确动作条件（止损线、目标价、失效条件） → 卖出提醒（动作级）
- 同时满足两类条件 → 卖出提醒（动作级优先于关注级）

#### 统一建议动作级别

| 动作级别 | 语义 | 适用场景 |
|---------|------|---------|
| `继续持有` | 无需动作 | 信号轻微，未达阈值 |
| `重点关注` | 需密切观察 | 指标恶化但未到动作点 |
| `考虑减仓` | 部分卖出降低风险敞口 | 减仓条件接近或达成 |
| `考虑止盈` | 可锁定收益 | 目标价达成、收益率达标 |
| `考虑止损` | 已触及止损条件 | 止损线跌破、失效条件触发 |
| `考虑清仓` | 持有逻辑已失效 | 严重基本面恶化 |

所有飞书通知和网页看板中的建议动作必须使用上述统一级别。

#### 信号阈值策略

> PRD 定义信号语义和当前版本默认阈值。以下默认阈值后续版本可在策略配置中调整。

---

#### 3.4.3 风险预警信号（关注级）

| 信号 | 触发条件（默认阈值） | 默认建议动作 |
|------|---------------------|-------------|
| 急跌预警 | 单日跌幅 ≥ 5% | `重点关注` |
| 阶段回撤 | 从阶段高点回撤 ≥ 15% | `重点关注` |
| 基本面恶化 | ROE 较最近研究时下降 ≥ 5 个百分点，或净利率由正转负，或负债权益比 ≥ 2.0 | `重点关注` |
| 杠杆风险升级 | 净债务/EBITDA ≥ 4.0，或利息覆盖倍数 ≤ 2.0 | `重点关注` |
| 技术面转弱 | 50日均线下穿200日均线（死叉），或当日成交量 ≥ 近20日均量 2 倍且收跌 | `重点关注` |
| 财报临近 | 距财报发布日 ≤ 3 个自然日 | `重点关注` |

> "重大事件"信号（重大负面新闻或风险事件）因数据来源和判定机制未定义，**降级到后续版本**。当前版本可通过持仓重研究（3.4.7）间接覆盖。

---

#### 3.4.4 卖出提醒信号（动作级）

| 信号 | 触发条件（默认阈值） | 默认建议动作 |
|------|---------------------|-------------|
| 止损触发 | 跌破研究报告中的止损条件，或跌破买入价 8% | `考虑止损` |
| 目标价达成 | 达到三情景目标价（保守/基准/乐观任一） | `考虑止盈` |
| 收益率达标 | 收益率 ≥ 20% | `考虑止盈` |
| 减仓条件达成 | 研究报告中的减仓触发条件达成 | `考虑减仓` |
| 估值过高 | 当前 PE 或 PB ≥ 基准情景目标价对应估值的 1.5 倍 | `考虑减仓` |
| 失效条件触发 | 研究报告中定义的失效条件被触发 | `考虑清仓` |
| 持有逻辑失效 | 最新研究结论从"值得投"变为"不值得投" | `考虑清仓` |
| 技术顶部信号 | RSI(14) ≥ 70 且 50日均线向下拐头 | `考虑减仓` |

---

#### 3.4.5 预警生命周期

每条预警/提醒都有明确的生命周期：

```
触发(triggered) → 已通知(notified) → 用户响应 → 终态
```

终态分为三种：
- **已确认(acknowledged)**：用户看到了，决定暂不操作。信号保持"活跃"直到清除条件满足
- **已处理(resolved)**：用户执行了对应操作（如已减仓、已止损），信号关闭
- **已失效(expired)**：触发条件不再成立，信号自动失效

生命周期规则：
- 触发后必须在当次通知窗口内发送飞书通知
- 用户可在网页看板中标记"已确认"或"已处理"
- 预警历史永久保留，用于复盘分析

#### 不同信号类型的失效规则

| 信号类型 | 自动失效规则 |
|---------|-------------|
| **风险预警（关注级）** | 触发指标恢复正常超过 3 个交易日，自动失效 |
| **价格类卖出提醒**（目标价达成、收益率达标、技术顶部） | 触发条件不再满足超过 3 个交易日，自动降级为"历史触达"状态。含义：曾经到过动作条件，但市场已回落 |
| **条件类卖出提醒**（止损、失效条件、持有逻辑失效） | **不自动失效**，必须由用户操作或手动关闭 |

> "历史触达"在网页看板中可见（区别于活跃预警），供用户复盘参考。

**失效判定口径**：
- "恢复正常"的判定方式：本轮 `AlertEngine.detect_signals()` 不再产生同类型信号，视为该轮已恢复。AlertManager 在每轮检测后，对未被重新触发的活跃预警累计恢复天数；被重新触发时恢复天数归零
- MVP 阶段每日运行一次检测，一轮 = 一个自然日 ≈ 一个交易日
- 活跃预警定义：status 为 `triggered` / `notified` / `acknowledged` 之一

#### 信号升级机制

风险预警可以升级为卖出提醒：
- 阶段回撤 → 止损触发：当同一轮检测中 `AlertEngine` 同时产生"阶段回撤"和"止损触发"信号时，AlertManager 将已有的"阶段回撤"预警标记为 `upgraded`，在新建的"止损触发"预警中通过 `upgrade_from_id` 关联。升级判定纯粹基于信号引擎结果，AlertManager 不自行读取行情
- 基本面恶化持续且重研究结论翻转 → 从"风险预警"升级为"持有逻辑失效"
- 升级时发送新的飞书通知，标注"由XX预警升级"

---

#### 3.4.6 多信号并发处理

同一只股票可能同时触发多个信号。处理规则：

**合并原则**：
- 同一交易日、同一只股票的多个信号，合并为一条飞书通知
- 通知中按动作级别从高到低排列所有活跃信号
- 整体建议动作取最高级别信号的建议

**优先级**（从高到低）：
1. `考虑清仓` > `考虑止损` > `考虑止盈` > `考虑减仓` > `重点关注` > `继续持有`

**示例**：某只股票同时触发"阶段回撤 15%"（`重点关注`）和"跌破止损线"（`考虑止损`），合并为一条通知，整体建议为 `考虑止损`，通知内容同时列出两个触发信号。

---

#### 3.4.7 持仓重研究

持仓股票因显著变化打破 10 天去重触发重研究时，完整产品动作如下：

**触发条件**（同 Step 3 去重例外）：
- 持仓股单日价格变动 ≥ 5%
- 技术面状态发生质变（上升→下降 或 下降→上升）

**产品动作**：
- 生成**新的飞书文档**（不覆盖旧文档），标题格式：`[股票代码] 公司名 - 重研究报告 (日期)`
- 持仓视图中的"最近研究结论"更新为最新结论
- 系统数据层（止损条件、目标价、失效条件等）**更新为最新研究值**
- 旧研究保留在研究历史中，可在个股详情页查看
- 飞书发送"重研究完成"通知，包含新文档链接和关键变化摘要

**新旧结论冲突时**：
- 以最新研究为准，系统数据层全量覆盖
- 如果新研究结论从"值得投"变为"不值得投"，同时触发"持有逻辑失效"卖出提醒

---

#### 3.4.8 收益率统计

用户可在网页看板中查看：
- 单只股票收益率（未实现 / 已实现）
- 总持仓市值和总成本
- 总未实现盈亏
- 已卖出交易的历史收益

计算口径见 3.3 节"成本与收益计算口径"。

#### 3.4.9 卖出记录

用户决定卖出后，手动录入卖出信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 股票代码 | 是 | 大写 |
| 卖出价格 | 是 | 美元 |
| 卖出日期 | 是 | 默认当天 |
| 卖出数量 | 是 | 股数 |
| 卖出原因 | 否 | 自由文本 |

系统行为：
- 卖出记录写入数据库
- 计算已实现盈亏（按加权平均成本法）
- 全部卖出 → 转入历史交易，关联的活跃预警自动关闭
- 部分卖出 → 更新持仓数量和成本，预警保留

#### 验收标准
- 用户打开飞书或网页，5 分钟内能判断所有持仓是否安全

- 条件类信号不自动失效
- 持仓重研究后系统数据自动更新
- 预警历史可追溯
- 买入/卖出记录完整，收益计算口径一致

每只股票至少展示：

| PE / PB / ROE | 关键估值和质量指标 |
| 策略命中原因 | 主策略名 + 关键命中条件（最多 3 条） |
| 研究状态 | 未研究 / 已研究(日期) / 降级研究(日期) / 研究失败 / 复用(日期) / 待研究 |

交互能力：

### 3.6 策略配置

#### 功能目标
用户可在网页看板中配置筛选策略，策略基于数据源可提供的指标构建。

#### 策略结构

一条策略由三部分组成：
- **筛选条件（Filter）**：从全市场中筛出候选股
- **排序规则（Ranking）**：对候选股排优先级
- **门槛条件（Gate）**：排除明显不合格的股票

#### 受控开放原则

策略配置是**受控开放能力**，不是把数据源全量字段裸露给用户：
- 只开放数据源稳定支持、业务上有明确意义、用户可理解的指标
- 后续根据需求逐步开放更多指标

#### 当前版本开放的指标分类

| 类别 | 包含指标 | 说明 |
|------|---------|------|
| 估值类 | PE、PB、PS、PEG | 价格是否合理 |
| 盈利能力类 | ROE、ROA、毛利率、营业利润率、净利率、ROIC | 赚钱能力 |
| 财务健康类 | 负债权益比、流动比率、速动比率、利息保障倍数 | 财务安全 |
| 成长类 | 营收增长率、净利润增长率、营业利润增长率、EPS增长率 | 增长动力 |
| 现金流类 | FCF Yield、OCF Yield、股息支付率 | 现金质量 |
| 市场类 | 市值、股价、成交量、Beta、股息率、涨跌幅、距52周高点跌幅 | 市场特征 |
| 分类过滤 | 行业、子行业、交易所、国家 | 范围限定 |

> 各指标的完整中英文名称、说明和示例值见附录 B。

**需要额外本地计算的指标**（数据源不直接支持）：
- 技术指标（MA、RSI、MACD）— 需获取历史价格后本地计算
- EV/EBITDA — 需获取企业价值和 EBITDA 后计算

#### 策略配置界面
- 查看、新建、编辑、启用/禁用、删除策略
- 不合法的配置（如 PE 最小值大于最大值）有提示
- 至少预置一条示例策略
- 策略修改后下次筛选生效

### 3.7 通知体验设计

#### 设计目标
用户每天在飞书中收到系统通知，不应感到被打扰或信息过载，而是高效获取所需信息。

#### 通知类型

| 事件 | 通知内容 | 频率 | 紧急度 |
|------|---------|------|--------|
| 每日筛选完成 | 命中股票数 + 每只一句话结论 + 报告链接 | 每日一次 | 常规 |
| 深度研究完成 | 股票代码、结论摘要、文档链接 | 事件触发 | 常规 |
| 买入确认 | 股票代码、价格、数量 | 事件触发 | 常规 |
| 风险预警 | 股票代码、预警类型、关键数值、建议动作 | 事件触发 | 紧急 |
| 卖出提醒 | 股票代码、提醒类型、关键数值、建议动作 | 事件触发 | 紧急 |
| 持仓重研究完成 | 股票代码、新文档链接、关键变化摘要 | 事件触发 | 紧急 |
| 系统运行失败 | 失败范围、错误摘要 | 事件触发 | 最紧急 |

#### 通知治理规则

**防打扰**：
- 同一股票同类事件 6 小时内不重复通知（严重度升级除外）
- 每日筛选结果合并为一条汇总通知
- 同一股票同日多个预警信号合并为一条通知（见 3.4.6）

**确保到达**：
- 失败通知优先级高于正常通知
- 通知发送失败必须可追踪，重试 3 次后记录错误
- 紧急通知（风险预警、卖出提醒）不受日常限流影响

#### 系统失败通知粒度

| 失败场景 | 通知方式 |
|---------|---------|
| 整个定时任务未启动 | 独立发送失败通知 |
| 数据源筛选失败（无结果返回） | 独立发送失败通知 |
| 单只股票研究失败 | 合并到筛选汇总通知中标注 |
| 飞书文档生成失败 | 合并到筛选汇总通知中标注 |
| 部分成功部分失败 | 汇总通知正常发送，逐只标注状态 |

#### 用户一天的通知体验

典型持仓 3-5 只股票的用户：
- **常规日（无异常）**：仅收到 1 条每日筛选汇总通知
- **有新发现时**：筛选汇总 + 深度研究完成通知
- **持仓有异常时**：对应股票的预警通知，每只股票至多 1 条（合并多信号）
- **最坏情况**：全部持仓同时异常，收到 3-5 条预警通知

> 设计目标：常规日 ≤ 2 条通知，异常日 ≤ 持仓数 + 2 条通知。

## 4. 深度研究能力

### 概述

系统通过 Perplexity API 对候选股票进行机构级深度研究。研究不是让 AI 写一篇漂亮的报告，而是回答 6 个核心问题（见场景二）。

### 研究方法论

研究框架按以下顺序推进，**不得先下结论再倒推理由**：

```
商业模式 → 财务报表 → 真实盈利 → 真实资产 → 杠杆风险 → 估值 → 交易决策
```

同时覆盖三个分析视角：
1. **卖方研究员**：行业、基本面、财报、竞争格局
2. **价值投资分析师**：真实资产、归一化盈利、安全边际
3. **交易员**：股价异动、催化剂、交易计划

### 两层输出设计

研究输出分为两层：
- **飞书文档层**：完整 Markdown 报告，用户阅读决策（见 3.2）
- **系统数据层**：从报告中提取结构化字段，供买后管理自动化使用（止损条件、目标价、失效条件等）

### 质量要求

当前版本以 3.2 第一阶段 6 项内容为**交付门槛**。附录 A 中的完整校验为**质量目标**：
- 未达交付门槛 → 不推送文档，标记研究失败
- 达到交付门槛但未达质量目标 → 推送文档，标记"部分内容缺失"
- 达到质量目标 → 完整推送

> 完整的研究规格（触发规则、输入要求、硬性约束、输出结构、质量校验规则、降级策略）见附录 A。

## 5. 非功能约束

### 5.1 运行频率
- 每日筛选：每个美股交易日运行一次（开盘后 5 分钟）
- 持仓监控：每个交易日至少刷新一次
- 非交易日不运行

### 5.2 数据源额度
- 数据源存在调用额度限制，系统在接近额度上限时需给出预警
- 每日预估消耗需合理规划：筛选 + 候选股详情 + 持仓刷新
- 具体额度限制和分配方案在技术设计文档中定义

### 5.3 数据保留
- 筛选记录、研究报告、交易记录：永久保留
- 日常快照：至少保留 90 天

### 5.4 单机运行
- 系统运行在本地 Mac 上
- 无需部署到服务器

| 每日自动筛选后 | 自动触发，受 10 天去重和每日 15 只上限约束 |
| 手动运行全流程 | 手动触发完整批次，受 10 天去重约束，受每日上限约束 |
| 手动指定单只股票 | 立即执行，不受去重和上限约束 |
| 持仓重研究 | 持仓股票出现显著变化时自动触发，可打破 10 天去重 |

显著变化的定义：
- 单日价格变动绝对值 ≥ 5%
- 技术面状态发生质变（由涨转跌或由跌转涨）

并发控制：
- Perplexity 研究请求需串行或限流执行，避免触发 API 限流
- 具体并发策略在技术设计文档中定义

### A.2 研究输入

每次研究必须基于一份明确的输入快照，不允许让模型自行补全背景。

**必要字段（缺失则不执行研究）**
- `symbol`：大写股票代码

**重要字段（缺失可继续，但需标注信息不完整）**
- `sector`
- `exchange`
- `price`
- `market_cap`
- `volume`
- `ratios`（结构化比率集合）

输入快照必须随研究记录保存，以便事后核验。

### A.3 研究方法论

研究不是让 AI 写一篇漂亮的报告，而是回答 6 个核心问题：

1. 这家公司到底有没有真家底？
2. 报表利润有多少水分？
3. 当前价格买入，对应的真实收益率/年大概是多少？
4. 安全边际来自哪里？是资产、现金流还是两者兼有？
5. 下跌时值不值得接？上涨后该不该追？
6. 如果用自己的钱投，应该怎么买、买多少、错了怎么退？

研究框架必须按以下顺序推进，**不得先下结论再倒推理由**：
```
商业模式 → 财务报表 → 真实盈利 → 真实资产 → 杠杆风险 → 估值 → 交易决策
```

#### 三个分析视角

研究同时覆盖三个视角：
1. **卖方研究员视角**：行业、基本面、财报、公告、指引、竞争格局
2. **价值投资分析师视角**：真实资产、有形净资产、归一化盈利、估值、安全边际
3. **交易员视角**：股价异动、催化剂、事件驱动、交易计划

---

### A.4 信息源优先级

研究结论的数据来源必须遵守以下优先级：

**A. 一级来源（必须优先）**
- 年报 / 20-F / 10-K
- 季报 / 10-Q / 6-K
- 8-K、正式公告、交易所公告、公司新闻稿
- 投资者演示材料、业绩新闻稿、电话会纪要

**B. 二级来源（补充使用）**
- SEC / 监管数据库
- 主流财经媒体（路透、彭博、雅虎财经等）
- 分析师评级、目标价更新

**C. 三级来源（仅辅助，不可单独支撑关键结论）**
- 第三方财经网站、聚合数据平台
- 卖方研报摘要
- 论坛、博客、社媒

规则：
- 财务数据、资产负债表、债务、回购、分红：必须来自一级来源
- 暴跌/暴涨原因：必须先查公告、财报、8-K/6-K，再补查媒体报道
- 一级与二级来源冲突时，以一级来源为准
- 关键数据无法在一级来源找到时，可用二级来源但须注明
- 仍找不到须写"未获取到可靠一级来源数据"

### A.5 研究硬性约束

以下约束不可违反，必须写入 Prompt 模板：

#### A.5.1 财务数据约束
- 分析窗口：最近 3 个完整财年 + 最近 4 个季度（或最近 1 年 + 最近季度）
- 所有收入、利润、现金流、债务、股本、商誉、无形资产等，优先引用法定披露文件

#### A.5.2 归一化盈利约束
- 不得直接把报表净利润视为真实盈利
- 必须识别并调整以下一次性项目（如适用）：
  - 资产出售收益/损失
  - 减值/减值转回
  - 重组费用
  - 并购整合费用
  - 诉讼/罚款/和解
  - 税项异常
  - 公允价值变动
  - 异常汇兑
  - 管理层标注为 non-recurring / one-off 的项目
- 必须输出"报表净利润 → 归一化净利润"的桥接表

#### A.5.3 资产真实性约束
- 必须拆分：现金、受限现金、应收、库存、固定资产、投资资产、商誉、其他无形资产
- 必须计算：
  - 有形资产 = 总资产 - 商誉 - 其他无形资产
  - 有形净资产 = 有形资产 - 总负债 - 少数股东权益
  - 每股有形净资产 = 有形净资产 / 稀释后股本
- 不得把商誉、品牌、客户关系等直接视为现金或可清算资产

#### A.5.4 杠杆与生存约束
- 必须给出：总债务、净债务、债务期限结构（至少未来 12 个月到期额）、利息支出、可动用流动性
- 至少计算：净债务/EBITDA、利息覆盖倍数
- 若未来 12 个月到期债务接近或超过流动性，必须把再融资风险写进结论，不得淡化

#### A.5.5 暴跌/暴涨原因约束
- 必须定位最近 1 个月内最大单日涨跌
- 必须先查当日及前后 48 小时内的：财报/业绩公告、8-K/6-K、公司新闻稿、监管文件
- 仍无明确触发时，只能写"未见公司层面明确触发，可能为流动性/情绪因素"
- 对每条原因结论，必须明确写是"已证实"还是"未证实"

#### A.5.6 估值约束
- 不得只用一个估值指标
- 至少使用两类不同方法中的两种：
  - 资产法（TBV、PB、NAV、重置成本）
  - 收益法（Normalized PE、EV/EBITDA）
  - 现金流法（FCF Yield、简化 DCF）
  - 市场法（可比公司估值）
- 必须输出：保守/基准/乐观三种情景的目标价区间及对应折价/溢价

#### A.5.7 缺失数据约束
- 不能静默省略缺失数据
- 在要点表中填 null，在正文中说明"未获取到可靠一级来源数据"及其对结论可靠性的影响

### A.6 Prompt 规格

Prompt 由两部分组成：
1. **系统提示词**：研究方法论、角色设定、硬性约束、输出结构要求（固定模板）
2. **用户消息**：股票代码 + 输入快照上下文

Prompt 模板治理：
- 模板需版本化管理
- 每次研究记录使用的模板版本
- 模板修改后应能通过版本区分历史结果

Prompt 禁止事项：
- 不得在没有公告/新闻证据时编造暴跌/暴涨原因
- 不得把管理层 adjusted profit 原样照搬成真实盈利
- 不得忽略少数股东权益、债务期限、股本稀释
- 不得用模糊措辞代替明确的 null/未证实标注
- 收到股票代码后直接开始分析，不重复讲解方法论

Perplexity 返回的完整报告，直接转为飞书文档。报告结构如下：

**第一阶段：极简综合要点表**

一张表格，覆盖以下字段（无数据填 null）：

| 字段 | 说明 |
|------|------|
| 股票代码 | |
| 公司名称 | |
| 当前股价 | |
| 每股有形净资产 | |
| 当前股价 / 每股有形净资产 | P/TBV 倍数 |
| 归一化 EPS | 剔除一次性项目后 |
| 归一化盈利收益率 | 归一化 EPS / 当前股价（%） |
| 净债务 / EBITDA | |
| 利息覆盖倍数 | |
| 商誉占总资产比例 | % |
| 无形资产占总资产比例 | % |
| 有形净资产是否为正 | true / false / null |
| 安全边际来源 | assets / cashflow / both / weak |
| 主要风险（最多 3 条） | |
| 建议买入区间 | 低 ~ 高 |
| 最大建议仓位 | % |
| 综合结论 | 值得投 / 不值得投 / 仅高风险偏好 |

表格之后，输出 3 句话结论：
- 第 1 句：公司的真实资产底线是什么
- 第 2 句：剔除水分后的真实盈利能力是什么
- 第 3 句：当前价格下，买入的核心逻辑和最大风险各是什么

**第二阶段：完整研究报告（7 章）**

**第一节：近期股价波动与直接催化**
- 最近 12-24 个月股价走势（涨跌幅、波动特征、年内高低点）
- 最近 1 个月最大单日涨跌：幅度、成交量变化（相对均量放大倍数）
- 触发原因（必须注明信息来源：公告/财报/媒体/"未见明确披露"）
- 触发类型判断：情绪驱动 / 基本面驱动 / 行业宏观驱动

**第二节：商业模式与业务结构**
- 公司如何赚钱（核心产品/服务/收费模式）
- 分业务板块表格：收入占比、同比增速、毛利率、EBIT 贡献、角色定位（现金牛/增长/问题）
- 产业链位置
- 主要客户与合作伙伴
- 护城河的真实高度（区分"真正的竞争优势"和"仅靠景气支撑"）

**第三节：财务质量与真实盈利**
- 核心财务趋势表：最近 3 个财年 + TTM（收入、毛利率、营业利润率、净利率、EPS、经营现金流、资本开支、自由现金流）
- 利润挤水桥接表：报表净利润 → 逐项调整 → 归一化净利润 → 归一化 EPS → 归一化 EBITDA
- 真实盈利收益率 = 归一化 EPS / 当前股价
- 盈利质量评估：经营现金流 / 归一化净利润比值；自由现金流是否持续为正

**第四节：真实资产与负债安全性**
- 资产结构表：现金/受限现金、应收、库存、固定资产、投资、商誉、其他无形资产（各项占总资产比 + 真实性评注）
- 有形净资产计算：有形资产、有形净资产、每股有形净资产
- 负债与杠杆表：总债务、有息债务、净债务、净债务/EBITDA、利息覆盖倍数、未来 12 个月到期债务、可动用流动性、再融资风险评级
- 安全性结论

**第五节：行业与竞争格局**
- 行业规模与增速（过去 3-5 年 CAGR，未来预期）
- 行业周期性：受宏观、利率、监管、替代品影响的敏感度（高/中/低）
- 主要竞争对手对比表：收入规模、市值、ROE、净利率、PE、PB
- 公司在行业中的位置
- 护城河真实高度

**第六节：估值与安全边际**
- 当前估值指标表：PE(TTM)、PE(归一化)、P/TBV、PS、EV/EBITDA(归一化)、FCF Yield、股息率，与行业均值对比
- 三情景估值表：

| 情景 | 假设 | 估值方法 | 倍数 | 目标价低 | 目标价高 | 相对当前折溢价 |
|------|------|---------|------|---------|---------|--------------|
| 保守 | | | | | | |
| 基准 | | | | | | |
| 乐观 | | | | | | |

- 当前市场定价最接近哪一情景的隐含假设

**第七节：投资结论与交易计划**
- 投资结论：值得投 / 不值得投 / 仅高风险偏好，及理由
- 交易计划表：

| 执行项目 | 价格 / 条件 / 仓位 |
|---------|-------------------|
| 持有期限 | |
| 最大仓位 | X% |
| 首次建仓区间 | X ~ Y |
| 二次加仓条件 | |

### A.8 质量校验

**A. 交付门槛（系统自动校验，不通过则不推送文档）**

对应 3.2 第一阶段 6 项内容：
1. 要点表存在且核心字段不全为 null
2. 3 句话结论不能缺失
3. 包含看多/看空逻辑
4. 包含关键风险与催化剂
5. 包含估值观点与目标价区间
6. 包含投资结论与操作建议

**B. 质量目标（系统自动校验，不通过则标记"部分内容缺失"，不阻塞交付）**
1. 包含利润挤水桥接表（第三节）
2. 包含有形净资产计算（第四节）
3. 包含三情景估值完整表（第六节）
4. 包含交易计划表（第七节）
5. 数据来源以一级来源为主

**C. 内容质量（供用户阅读判断，不做系统校验）**
1. 归一化调整是否合理
2. 有形净资产计算是否准确
3. 三情景估值假设是否现实
4. 投资结论与前面分析是否自洽
5. 暴跌/暴涨原因是否有证据支撑

### A.9 失败与降级

- Perplexity 不可用时，必须有降级研究行为
- 降级报告可以简化，但至少要满足交付门槛（A.8 层 A）
- 降级报告如果也无法满足交付门槛，则标记为研究失败
- 用户必须能分辨"完整研究"、"降级研究"和"研究失败"
- 原始失败原因必须保留，不得静默吞掉
- 降级报告在飞书文档标题中加注"[降级]"标识

---

### A.10 落库与追溯

每次研究必须保存：
- 完整报告原文
- 提取的全部结构化字段
- 输入快照
- 全部元数据字段

用户必须能知道：
- 这份研究基于哪只股票、什么时间做的
- 结论是什么（值得投 / 不值得投）
- 安全边际来源是什么
- 建议的买入区间和止损条件
- 是完整研究、降级研究还是研究失败
- 飞书文档在哪里
