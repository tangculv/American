# 美股选股体系开发计划

## 1. 计划目的

本文件是当前项目的唯一开发推进主线，承接 `docs/PRD-BUSINESS.md`（v3）和 `docs/TECHNICAL-DESIGN.md`。

后续开发必须围绕本计划执行：
- 所有任务都要对应 PRD 章节和技术设计模块
- 所有任务都要有验收标准
- 所有任务都要经过测试
- 最终交付必须回到 PRD 做还原验收

## 5. Phase 2：买后管理核心

> 目标：实现信号检测 + 预警生命周期 + 持仓监控集成。这是系统最核心的新增能力。

### Task 3：信号检测引擎

- **对应 PRD**：3.4.3 风险预警信号、3.4.4 卖出提醒信号
- **对应技术设计**：§9.2 信号检测引擎
- **优先级**：P0
- **详细描述**：
  - 新增 `alert_engine.py`
  - 实现 6 种风险预警信号：急跌预警（≥5%）、阶段回撤（≥15%）、基本面恶化（ROE 降≥5pp）、杠杆风险升级（Net Debt/EBITDA≥4）、技术面转弱（死叉）、财报临近（≤3天）
  - 实现 7 种卖出提醒信号：止损触发（≤-8%或研究止损条件）、目标价达成（三档）、收益率达标（≥20%）、失效条件触发、持有逻辑失效、技术顶部信号（RSI≥70+均线拐头）、估值过高
  - 每种信号需携带：type、level（warning/action）、action（6 级）、value、threshold、detail
- **修改文件**：新增 `alert_engine.py`
- **验收标准**：
  - 每种信号的触发/不触发边界测试通过
  - 信号输出结构完整（type/level/action/value/threshold/detail）
  - 无数据时不崩溃（graceful degradation）
  - 单测覆盖全部 13 种信号
- **当前状态**：未开始

---

### Task 4：预警生命周期管理

- **对应 PRD**：3.4.5 预警生命周期、3.4.6 多信号合并
- **对应技术设计**：§9.3 预警生命周期管理
- **优先级**：P0
- **详细描述**：
  - 新增 `alert_manager.py`
  - 实现生命周期状态流转：triggered → notified → acknowledged / resolved / expired / historical_reached / upgraded
  - 实现信号升级：阶段回撤触及止损线 → 升级为止损触发
  - 实现差异化失效规则：
    - 风险预警：恢复正常 3 个交易日自动 expired
    - 价格类卖出提醒（目标价/收益率/技术顶部）：不满足 3 个交易日 → historical_reached
    - 条件类卖出提醒（止损/失效/逻辑失效）：不自动失效
  - 实现多信号合并通知：同股多信号合并为一条，取最高动作级别
  - 动作优先级排序：考虑清仓(6) > 考虑止损(5) > 考虑止盈(4) > 考虑减仓(3) > 重点关注(2) > 继续持有(1)
- **修改文件**：新增 `alert_manager.py`
- **验收标准**：
  - 生命周期状态流转正确
  - 价格类 vs 条件类失效规则区分正确
  - 升级链路正确（回撤→止损）
  - 合并通知取最高动作级别
  - 单测覆盖：每种状态转换、升级、失效、合并
- **当前状态**：未开始

## 12. 最终交付标准

项目最终交付时，不以"代码完成"为标准，而以"还原业务 PRD"为标准。

最终必须回答：
1. PRD 中定义的核心功能是否都已实现
2. 使用者是否能按 PRD 描述使用系统
3. 技术实现是否稳定可验证
4. 开发计划中的任务是否完成或明确标注边界

只有回到 `docs/PRD-BUSINESS.md` 逐条核验通过，才算真正完成交付。
